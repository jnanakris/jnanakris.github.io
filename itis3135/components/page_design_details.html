Page Design Details

1. Home Page – index.html

  • Purpose: Introduce the platform and connect travelers seeking help with co-passengers.
  • Audience: All users
  • Content: Overview, CTA buttons, testimonials
  • Data Entry: No
  • Validations: N/A
  • Interactivity: CTA buttons
  • Actions: Navigate to login/register
  • Notes: Mobile-friendly

2. About Page – about.html

  • Purpose: Explain system, safety tips, FAQs
  • Audience: General visitors
  • Content: FAQs, step-by-step guide
  • Data Entry: No
  • Validations: N/A
  • Interactivity: Expandable FAQ (jQuery UI)
  • Actions: Navigation only
  • Notes: Use accordion

3. My Requests – requests.html

  • Purpose: Display user’s submitted help requests
  • Audience: Customers
  • Content: List of requests, edit/delete options
  • Data Entry: No
  • Validations: N/A
  • Interactivity: Edit/Delete buttons
  • Actions: Redirect to edit, delete confirms
  • Notes: Badge shows count

4. My Matches – matches.html

  • Purpose: Show volunteer matches and reviews
  • Audience: Customers & volunteers
  • Content: Match list, review form
  • Data Entry: Yes
  • Validations: Required review fields
  • Interactivity: Buttons for review/message
  • Actions: Submit review or message
  • Notes: Auto-hide after flight

5. Post a Request – post.html

  • Purpose: Submit new help request
  • Audience: Customers
  • Content: Flight and help info
  • Data Entry: Yes
  • Validations: Required fields, datepicker
  • Interactivity: jQuery UI, dropdowns
  • Actions: Submit → Confirm → Redirect
  • Notes: Simulate $1 token

6. Profile – profile.html

  • Purpose: View/edit user profile
  • Audience: All users
  • Content: Name, virtual ID, preferences
  • Data Entry: Yes
  • Validations: Email format, required fields
  • Interactivity: Toggle preferences
  • Actions: Save profile changes
  • Notes: Optional photo upload

7. Settings – settings.html

  • Purpose: Account settings and preferences
  • Audience: All logged-in users
  • Content: Notification, reset password, delete account
  • Data Entry: Yes
  • Validations: Confirmation for delete
  • Interactivity: Checkboxes, reset button
  • Actions: Save settings or log out
  • Notes: Add accessibility toggle
